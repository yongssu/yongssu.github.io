redis客户端使用方法总结

Posted on 2019-01-19

简介

本文主要讲解spring-boot&java下redis客户端的使用以及python下python客户端的使用。

spring-boot&java

SpringDataRedis项目通过使用一个键值样式的数据存储，将核心Spring概念应用到解决方案的开发中。提供了一个“模板”，作为发送和接收消息的高级抽象。与Spring框架中的JDBC支持类似。

Spring Data Redis提供了从Spring应用程序轻松配置和访问Redis的功能。它提供了与存储交互的低级和高级抽象，从而使用户不必担心基础设施问题

Spring ReDIS需要ReDIS 2.6或以上，Spring Data Redis支持与 Lettuce 和 Jedis 集成,目前Lettuce对于redis集群支持的比较完备。我们的样例将以lettuce-io作为讲解。

Spring Data Redis特性：

  • Spring Data Redis 可以支持对redis的基本数据读取与写入
  • Spring Data Redis 可以实现主节点写，副本节点读
  • Spring Data Redis 支持Sentinel，用于处理高可用Redis

RedisTemplate处理对象
redisTemplate是Redis模块的中心类，提供了高级抽象与redis进行交互。同时RedisConnection提供了一个低级方法处理和返回二进制数据(byte arrays). RedisTemplate关注学裂化和管理，使得用户无需处理这些细节。

此外，该Template还提供操作视图（根据redis命令引用中的分组），这些视图提供丰富的、通用的接口，用于处理特定类型或特定键（通过键绑定接口），如下表所述。

具体操作视图描述表格

配置后，模板是线程安全的，可以跨多个实例重用。

RedisTemplate使用基于Java的序列化器进行大部分操作，这意味着模板的写入或读取的任何对象都是java序列化，并通过Java反序列化的。可以改变template的序列化机制并且redis提供了几个接口，在org.springframework.data.redis.serialize包下。

Spring redis Data还抽象了如下redis机制功能:

  • Redis Messaging(Pub/Sub)
  • Redis Transactions机制
  • Redis pipeline
  • Redis lua脚本eval evalhash
  • Redis cluster支持
  • 与Spring CacheManage结合
  • Redis Repositories作为存储
